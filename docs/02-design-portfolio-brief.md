# NIXIANG 设计作品集产品 Brief

## Design Read

面向潜在客户、设计负责人和招聘方的设计师/小型工作室作品集。视觉语言是一座可探索的数字展览，信息结构保持编辑式清晰。参考 Bruno Simon 的“空间即导航”，但不复制车辆、地图、配色、文案或品牌。

## 设计旋钮

- DESIGN_VARIANCE: 8
- MOTION_INTENSITY: 7
- VISUAL_DENSITY: 3

## 核心命题

传统作品集让人浏览缩略图。本方案让访客进入一个“设计现场”，先看见作者的审美判断，再选择深入案例。

站点提供两个平等入口：

1. 展岛模式：适合第一次访问和感受品牌。项目以空间化展台出现，可拖动、键盘移动或直接点击。
2. 索引模式：适合招聘方和回访用户。按项目、年份和类别快速扫描，键盘和屏幕阅读器均可访问。

两个模式共享同一份项目数据，任何项目都能通过普通链接打开，不把内容锁在 Canvas 或动效里。

## 工作标题

NIXIANG / 设计现场

这是原型名称，后续替换为真实姓名或工作室品牌。

## 目标用户

### 潜在客户

需要在 60 秒内确认风格、能力范围和合作入口。

### 设计负责人

关注问题定义、系统思考、过程质量与最终交付，不只看效果图。

### 招聘方

需要快速扫描项目类型、职责、年份和案例深度，并能直接下载简历或联系作者。

## 信息架构

### 首页

- 单行导航：品牌、作品、方法、关于、联系。
- 模式切换：展岛 / 索引。
- Hero：一句明确定位、一个作品入口、原创展览主视觉。
- 精选项目：4 个真实案例槽位。
- 方法：研究、系统、呈现三个能力段，使用非等宽布局。
- 关于：短简介、合作范围、当前状态。
- 联系：一个统一 CTA“开始合作”。

### 项目详情

- 项目标题、年份、类别、职责。
- 一句结果导向摘要。
- 全宽主视觉。
- 背景与问题。
- 过程：洞察、系统、关键迭代。
- 交付：品牌、界面、空间、动效等。
- 结果与复盘。
- 下一个项目。

### 独立访问路径

- /work 或首页 #work：全部项目索引。
- /work/:slug：单个案例。
- /about：详细履历和能力。
- /contact：联系表单或邮件入口。

原型阶段采用单页与可访问项目详情弹层，正式版再切换为路由页面。

## 首页交互

### 展岛模式

- 主视觉是一张原创 16:10 设计展览图。
- 4 个项目节点围绕主视觉形成不规则构图。
- 指针移动只产生轻微景深和位移，不改变内容位置。
- 点击节点打开项目详情。
- 箭头键可在节点间移动，Enter 打开。

### 索引模式

- 左侧为项目标题、年份和类别。
- 右侧保留一个大图预览。
- Hover 与 Focus 都能切换预览。
- 移动端直接变成单列图文列表，不依赖 hover。

### 动效目的

- Hero 入场：建立视觉层级。
- 模式切换：解释空间视图和索引视图之间的状态变化。
- 项目预览切换：提供选择反馈。
- 项目详情打开：保持选中项目的空间连续性。

所有动效遵守 prefers-reduced-motion。没有滚动劫持，没有自定义光标，没有持续无意义循环。

## 视觉系统

### 主题

主视觉为深色。通过 CSS 变量提供完整浅色替代，并跟随系统设置。页面内部不做明暗主题分段翻转。

### 颜色

- 深色背景：#111214
- 深色表面：#1a1c1f
- 深色文字：#f1f0ec
- 浅色背景：#f2f1ed
- 浅色表面：#ffffff
- 浅色文字：#17181a
- 唯一强调色：#ff5c4d

强调色只用于主要 CTA、选中状态、项目编号和关键交互反馈。

### 字体

- Display：Arial Narrow / system condensed fallback，宽度收紧、字重 700。
- Body：Arial / system sans。
- Metadata：ui-monospace。

正式版应替换为有授权的自托管字体，并设置 font-display: swap。

### 形状

- 卡片与图片统一 14px。
- 按钮统一 999px，仅按钮使用 pill。
- 输入与弹层使用 14px。
- 不混用随机圆角。

### 图像

- Hero：原创生成的空间设计展览主视觉。
- 项目：原型使用明确标记的真实图片占位，正式版必须替换为项目原图。
- 不使用借来的 Bruno Simon 截图作为站点内容。

## 示例项目数据

### Rive Coffee

- 类型：品牌系统 / 包装
- 年份：2026
- 摘要：为一家城市烘焙品牌建立可扩展的包装与门店视觉系统。

### Muse

- 类型：数字展览 / Web
- 年份：2025
- 摘要：将馆藏叙事转译成可探索、可检索的数字展览体验。

### Northline

- 类型：城市导视 / Identity
- 年份：2025
- 摘要：用一套可复制的标识语言连接交通、街区和公共空间。

### Drift Bank

- 类型：产品设计 / Motion
- 年份：2024
- 摘要：重构移动金融的关键路径，并用动效解释状态与风险。

这些名称和内容是原型占位，不应被当作真实履历发布。

## 内容原则

- 首页只展示 4 个精选项目。
- 每个摘要不超过 32 个中文字符。
- CTA 只有三个意图：查看作品、切换模式、开始合作。
- 不编造客户数据、奖项或增长数字。
- 不用“赋能、重塑、颠覆、无缝”等空泛词。

## 可访问性

- 所有项目节点使用 button 或 link，不使用 click-only div。
- 图标按钮必须有 aria-label。
- 展岛模式不是唯一访问路径。
- 焦点样式使用强调色并保持 3:1 以上对比。
- Body 文本达到 WCAG AA。
- 支持 Escape 关闭详情。
- 弹层使用原生 dialog，打开后正确管理焦点。
- 动效支持 reduced motion。

## 移动端

- 导航保持单行，次级链接折叠到菜单。
- Hero 改为纵向：文案在上，主视觉在下。
- 展岛节点不做自由拖拽，改为横向 scroll-snap。
- 索引模式为默认推荐路径。
- 所有主要 CTA 在首屏可见。
- 使用 min-height: 100dvh，避免 iOS 地址栏跳动。

## 验收标准

- 1440 × 900 首屏不滚动即可看到主 CTA。
- 390 × 844 不横向溢出。
- 键盘可完成模式切换、项目选择、详情打开与关闭。
- reduced motion 下不出现漂浮、缩放或过渡延迟。
- 所有图片有宽高和替代文本，CLS 目标低于 0.1。
- 页面没有 em dash 字符、装饰状态点、滚动提示和重复 CTA 意图。
